# Classe inversée sur les arbres et les dictionnaires

Durée : 3 heures

## Objectifs

- Comprendre les concepts d'arbre binaire et de dictionnaire
- Rédiger un document en markdown
- Implémenter un arbre binaire et un dictionnaire en Python
- Mettre en pratique les demandes de tirage (pull requests) et les commentaires sur GitHub

## Instructions

### Étape 1 : Répartition de la classe (5 minutes)

- La classe sera divisée en deux : une moitié travaille sur les arbres binaires, l'autre moitié sur les dictionnaires.
- Pour chaque moitié, vous formez des groupes de 2 à 3 personnes. Chaque groupe
  fait une duplication de ce dépôt et travaille sur sa propre branche. Une
  personne dans le groupe devra faire une demande de tirage (pull request) pour
  que les autres puissent voir votre travail.

> [!NOTE]
> Vous devez choisir un nom pour votre groupe, qui sera utilisé pour nommer votre dossier de travail.


### Étape 2 : Rédaction du document markdown (30 minutes)

- Chaque groupe doit rédiger un document markdown expliquant les concepts d'arbre binaire et de dictionnaire, selon le sujet assigné. Votre document doit inclure :
  - Une définition claire de la structure de données
  - Les opérations principales et leurs noms standards
  - Les avantages et inconvénients de chaque structure
  - Des exemples d'utilisation en sciences ou en informatique

> [!IMPORTANT]
> Votre document markdown doit être dans le dossier `arbre/<nom-du-groupe>` ou `dictionnaire/<nom-du-groupe>` selon votre sujet, et doit être nommé `README.md`.

### Étape 3 : Implémentation en Python (40 minutes)

- Chaque groupe doit implémenter la structure de données assignée (arbre
  binaire ou dictionnaire) en Python sous forme d'une classe. Votre
  implémentation doit inclure les opérations principales sous forme de méthodes
  de classe. Assurez-vous que votre code est bien documenté et facile à
  comprendre.

**Votre implémentation Python doit être dans le même dossier que votre document markdown, et doit être nommée `implementation.py`.**

### Étape 4 : Demande de tirage (pull request) (5 minutes)

- Une fois que votre groupe a terminé la rédaction du document markdown et
  l'implémentation en Python, une personne du groupe doit faire une demande de
  tirage (pull request) pour que les autres puissent voir votre travail.

### Étape 5 : Révision de l'autre moitié de la classe (40 minutes)

- Si vous travaillez sur les arbres binaires, vous devez réviser le travail de l'autre
  moitié de la classe qui travaille sur les dictionnaires, et vice versa. Fournissez
  des commentaires constructifs sur leur document markdown et leur
  implémentation en Python dans la demande de tirage (pull request).

### Étape 6 : Mise à jour de votre demande de tirage (pull request) (30 minutes)

- Après avoir reçu des commentaires de l'autre moitié de la classe, mettez à
  jour votre document markdown et votre implémentation en Python en fonction des
  commentaires reçus.

### Étape 7 : Acceptation de la demande de tirage (pull request) (5 minutes)

- Votre professeur va accepter votre demande de tirage (pull request) une fois
  que vous avez mis à jour votre travail. Votre travail sera alors visible sur
  la branche principale du dépôt.
